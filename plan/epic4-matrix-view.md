### E4-S2 — 매트릭스 뷰 4사분면 투두 배치
> 상태: ✅ 완료

**사용자 스토리**  
매트릭스 뷰에서 투두가 긴급도·중요도에 따라 4개 사분면에 자동으로 배치된다.

**구현 세부사항**
- 사분면 분류 기준
  - **긴급+중요** (1사분면): `urgency >= 2` AND `importance >= 2`
  - **긴급+비중요** (3사분면): `urgency >= 2` AND `importance` is null or `importance < 2`
  - **비긴급+중요** (2사분면): `urgency` is null or `urgency < 2` AND `importance >= 2`
  - **비긴급+비중요** (4사분면): `urgency` is null or `urgency < 2` AND `importance` is null or `importance < 2` (단, 둘 다 null이 아닌 경우)
- 각 사분면에 사분면 이름과 설명 레이블 표시
- 투두가 없는 사분면도 빈 상태로 표시 (빈 사분면 안내 문구)
- 현재 필터(전체/진행 중/완료)가 각 사분면에 적용됨

**완료 기준**
- 긴급도·중요도 설정된 투두가 올바른 사분면에 배치 확인
- 빈 사분면에 안내 문구 표시 확인
- 필터 적용 시 각 사분면 투두 수 변화 확인

**체크리스트**
- [x] 사분면 분류 함수 구현 (`categorizeTodos` 순수 함수)
- [x] `todo-utils.ts`에 분류 함수 추가 및 단위 테스트 작성
- [x] 각 사분면 레이블/설명 표시
- [x] 빈 사분면 안내 문구 표시
- [x] 필터 연동 확인

### E4-S3 — 미분류 투두 별도 영역 표시
> 상태: ✅ 완료

**사용자 스토리**  
긴급도와 중요도가 모두 설정되지 않은 투두는 4사분면 그리드 외부의 "미분류" 섹션에 표시된다.

**구현 세부사항**
- 미분류 조건: `urgency IS NULL` AND `importance IS NULL`
- 매트릭스 그리드 하단에 "미분류" 섹션 별도 표시
- 미분류 투두가 없으면 섹션 미표시
- 현재 필터가 미분류 섹션에도 적용됨

**완료 기준**
- 긴급도·중요도 미설정 투두가 "미분류" 섹션에 표시 확인
- 미분류 투두 없을 때 섹션 미표시 확인
- 필터 적용 시 미분류 섹션 투두 수 변화 확인

**사용자 스토리**  
매트릭스 뷰에서도 투두 완료 체크박스 토글과 삭제가 목록 뷰와 동일하게 동작한다.

**구현 세부사항**
- `TodoMatrix`에서 `TodoItem` 컴포넌트 재사용 → 완료 토글·삭제 핸들러 그대로 전달
