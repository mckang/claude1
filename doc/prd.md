# PRD — 한국어 Todo 앱

## 1. 개요

| 항목 | 내용 |
|---|---|
| 프로덕트명 | 한국어 Todo 앱 |
| 버전 | 2.0 |
| 최종 수정 | 2026-04-08 |
| 배포 URL | https://prac1-flame.vercel.app |

### 목적
개인 할 일 관리를 위한 웹 애플리케이션. 사용자 인증을 통해 각자의 투두를 안전하게 저장하고 관리할 수 있다.

---

## 2. 사용자 스토리

### 인증
- 이메일·비밀번호로 회원가입/로그인할 수 있다.
- Google 계정으로 소셜 로그인을 할 수 있다.
- 로그아웃 후에는 자신의 데이터에 접근할 수 없다.
- 비인증 상태에서 `/` 접근 시 `/auth`로 자동 리다이렉트된다.

### 투두 관리
- 투두를 마크다운 형식으로 작성하고 렌더링된 형태로 볼 수 있다.
- 투두에 파일·이미지를 첨부할 수 있다.
- 기존 투두 텍스트를 인라인 편집할 수 있다.
- 투두를 체크하여 완료 처리하거나 취소할 수 있다.
- 투두를 개별 삭제할 수 있다.
- 완료된 투두를 일괄 삭제할 수 있다.
- 페이지를 새로고침해도 투두 목록이 유지된다.

### 긴급도·중요도 (아이젠하워 매트릭스)
- 투두에 긴급도(높음/보통/낮음)를 설정할 수 있다.
- 투두에 중요도(높음/보통/낮음)를 설정할 수 있다.
- 목록 뷰와 매트릭스 뷰(2×2) 두 가지 형태로 투두를 볼 수 있다.

### 마감일
- 투두에 마감일(날짜)을 설정하거나 해제할 수 있다.
- 마감일이 지난 투두는 강조 표시된다.

### 필터 & 통계
- 전체 / 진행 중 / 완료 탭으로 투두를 필터링할 수 있다.
- 진행 중·완료 건수와 전체 진행률을 한눈에 볼 수 있다.

### Google Calendar 연동 (Google OAuth 사용자 전용)
- 마감일이 있는 투두를 Google Calendar 이벤트로 등록할 수 있다.
- 앱에서 투두를 삭제하면 연동된 캘린더 이벤트도 삭제된다.
- 캘린더에서 이벤트를 삭제하면 연동된 투두도 삭제된다.
- 연동된 투두는 캘린더 아이콘으로 표시된다.

## 3. 기능 요구사항

### F-01. 회원가입
- 이메일과 6자 이상 비밀번호 입력 후 회원가입
- 가입 후 이메일 확인 메시지 표시

### F-02. 로그인
- 이메일/비밀번호 로그인
- Google OAuth PKCE 플로우 로그인

### F-03. 투두 CRUD
| 동작 | 트리거 | 동작 방식 |
|---|---|---|
| 추가 | Enter 키 또는 추가 버튼 | Supabase insert + 낙관적 업데이트 |
| 텍스트 편집 | 텍스트 더블클릭 또는 편집 버튼 | 인라인 편집, Enter/blur 저장, Escape 취소 |
| 완료 토글 | 체크박스 클릭 | Supabase update + 낙관적 업데이트 |
| 삭제 | 삭제 버튼 클릭 | Supabase delete + 캘린더 이벤트 연동 삭제 |
| 완료 일괄 삭제 | "완료 삭제" 버튼 클릭 | Supabase delete (done=true 전체) |

### F-04. 마크다운·첨부 지원
- 투두 본문에 마크다운 문법 사용 가능
- 렌더링된 마크다운과 원문 편집 모드 전환
- 파일·이미지 첨부 (Supabase Storage 저장, 투두당 최대 5개)
- 이미지는 인라인 미리보기, 그 외 파일은 다운로드 링크 표시

### F-05. 긴급도·중요도 설정
- 긴급도: 높음(3) / 보통(2) / 낮음(1), 미설정 가능
- 중요도: 높음(3) / 보통(2) / 낮음(1), 미설정 가능
- 배지(Badge) 형태로 투두 아이템에 표시

### F-06. 마감일 설정
- 날짜 피커로 마감일 설정·해제
- 마감일 초과 시 빨간 강조 표시

### F-07. 뷰 전환
- **목록 뷰** (기본): 기존 투두 리스트
- **매트릭스 뷰**: 긴급도(X축) × 중요도(Y축) 2×2 그리드
  - 4사분면: 긴급+중요 / 긴급+비중요 / 비긴급+중요 / 비긴급+비중요
  - 미설정 투두는 별도 영역에 표시

### F-08. 필터
- `all` (기본) / `active` / `done` 세 가지 뷰

### F-09. 진행률 표시
- 상단 진행 바: `완료 수 / 전체 수 * 100%`
- 통계 카드: 전체, 진행 중, 완료 건수

### F-10. 반응형 UI
- `lg` 이상: 사이드바 항상 표시
- `lg` 미만: 햄버거 버튼으로 사이드바 오버레이 전환

### F-11. Google Calendar 연동
- 조건: Google OAuth로 로그인한 사용자만 활성화
- 마감일이 있는 투두 → 캘린더 이벤트 생성 (단방향 트리거, 사용자 선택)
- 투두 삭제 → 연동된 캘린더 이벤트 자동 삭제
- 캘린더 이벤트 삭제 → 연동 투두 삭제 (Google Pub/Sub 웹훅 수신)
- Google access token: Supabase OAuth provider token 활용

## 4. 비기능 요구사항

| 분류 | 요구사항 |
|---|---|
| 보안 | RLS(Row Level Security)로 사용자 데이터 격리 |
| 성능 | 낙관적 업데이트로 UI 응답성 확보 |
| 접근성 | aria-label, aria-invalid, disabled 상태 정확히 반영 |
| 유지보수 | 비즈니스 로직을 순수 함수(todo-utils.ts)로 분리·테스트 |
| 배포 | Vercel 자동 배포 (main push) |
| 저장소 | Supabase Storage (첨부 파일, 사용자별 버킷 격리) |
